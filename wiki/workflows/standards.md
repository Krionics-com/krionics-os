# Workflow Documentation Standards

Use this structure for any workflow doc in wiki/workflows/.

Required sections
- Purpose: What problem the workflow solves.
- Scope: What is included and excluded.
- Inputs: Files, tools, or sources required.
- Outputs: Files produced or updated.
- Steps: Ordered steps with clear checkpoints.
- Checks: Validation or review steps.
- Artifacts: Index and log updates.

Naming
- workflow-<name>.md for reusable workflows.
- YYYY-MM-DD-<name>.md for dated runs or reports.

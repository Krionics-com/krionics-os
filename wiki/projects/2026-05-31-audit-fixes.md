## Architectural Changes

### Outbound Pipeline Now Runs Automatically

Before: "Launch Outbound" set a flag with no downstream effect. Apollo only ran on manual POST /api/apollo/import.

After:
- New worker `outbound-scheduler` runs hourly via BullMQ repeatable job.
- Scans `clients WHERE outbound_active = true`, checks `pull_cadence` vs `last_apollo_pull_at`, enforces `max_total_leads` and `threshold_min` ceilings.
- For each eligible client, calls `mapIcpToApolloSearchParams(client.config, client.apollo_config)` to translate ICP into Apollo search params, validates, and enqueues `apolloImportQueue`.
- Updates `clients.last_apollo_pull_at` after dispatch.

### Pre-Launch Validation Gate

Before: Launch button always clickable.

After: `POST /api/dashboard/clients/[slug]/launch-outbound` validates a 6-item preflight checklist server-side. Returns the checklist and missing items. UI fetches the checklist via GET, disables the button until ready, surfaces missing items as an inline list.

Checklist:
1. Cal.com link set
2. ICP configured (titles, industries, or geographies)
3. Sequence has at least 2 steps
4. Instantly Campaign ID set
5. At least one From Email
6. Review mode selected

### AI Prompts Now Override-Capable

Before: All prompts hardcoded in `prompt-builder.ts`. Edits required deploys.

After:
- Migration 20260531000003 seeds 6 global rows in `ai_prompts` (one per invocation type).
- New `loadPromptOverride(slug, clientId)` helper with 60s cache. Client-specific overrides preferred, falls back to global.
- AIProvider interface accepts optional `PromptOverride` (system_prompt, model, temperature, max_tokens) per method.
- sequence-generation worker reads override before calling generateSequence. Pattern is ready to roll out to other workers.

### Pipeline Metrics Replace Deprecated `active_campaigns`

Before: clients list/profile JOINed deprecated `campaigns` table — always returned 0 for new clients.

After: Real counts derived from `leads` lead_status: `leads_in_pipeline`, `leads_sending`, `replies_received`. Profile KPI cards: Leads in Pipeline / Sending / Replies / Meetings Booked.

### Real AI Cost Tracking

Before: stats API returned hardcoded $42.50. AI analytics returned `|| 0.42` fallback + random chart baselines.

After: All values pulled from `ai_invocations`. Per-client cost (today/7d/30d) shown on the client profile Overview tab.

### Infra Pages No Longer Show Mock Data

Before: `infra/inboxes` and `infra/domains` injected hardcoded `baseInboxes` array (krionics.com, krionics-biz.com, etc.) when DB returned no rows. Reputation/DNS/warmup all generated by deterministic hash of seed string. Misled operators into trusting fake numbers.

After: Real `email_events` aggregates only. Reputation/DNS/warmup return `null` until a real inbox-monitoring integration is in place.

## V2 Items (Deferred)

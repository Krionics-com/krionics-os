# Source Summary: LLM Wiki Idea

Summary
- Proposes a persistent, LLM-maintained wiki as a compiled knowledge layer between raw sources and queries.
- Emphasizes accumulation: new sources update existing pages, cross-links, and contradictions once, not per query.
- Recommends index.md and log.md as navigational primitives; optional search tools can be added later.

Key points
- Raw sources are immutable; the wiki is maintained by the LLM; the schema guides behavior.
- Ingest, query, and lint are the core operations.
- The human curates sources and questions; the LLM handles maintenance and cross-referencing.

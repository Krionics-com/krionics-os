# Architecture Documentation Standards

Use this structure for any architecture doc in wiki/architecture/.

Required sections
- Context: System or project background.
- Goals and non-goals: What the architecture must and must not do.
- Constraints: Technical, legal, or operational limits.
- Overview: High-level description of the system.
- Components: Major subsystems and responsibilities.
- Data flows: Inputs, outputs, and transformations.
- Interfaces: APIs, protocols, and contracts.
- Storage: Data stores and retention rules.
- Security and privacy: Threat model, controls, and data handling.
- Observability: Logs, metrics, traces, and alerts.
- Failure modes: Expected faults and mitigation.
- Decisions: Links to decision records, if any.
- Change log: Date-stamped revisions.
